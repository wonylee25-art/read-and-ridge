# Read & Ridge — 프로젝트 개요

## 컨셉

**산책또산책**(영문명: Read & Ridge)은 내가 읽는 책을 픽셀 아트 산으로 시각화하는 독서 기록 웹앱이다. (내용 출처: `docs/사용법.md` "1. 산책또산책이란", 2026.07.12 동기화)

- 책 한 권 = 산 하나. 책의 두께(총 페이지 수)가 산의 크기를 결정한다.
- 읽은 만큼 캐릭터(마스코트 "마루")가 산을 오른다. 페이지를 업데이트하면 실시간으로 위치가 올라간다.
- 완독하는 순간 산 정상에서 폭죽이 터지고 캐릭터가 양팔을 들고 춤추는 완등 세레모니가 재생된다. 이후 정상에 깃발이 잠깐 남아있다가, 그 책은 완등기록으로 옮겨가 쌓인다.
- 같은 책을 또 사는 걸 막기 위해 "소장 중" 여부를 별도로 관리하고, 이미 등록된 책을 중복 등록하려 하면 경고한다.

**소장 여부와 열람 상황은 서로 다른 두 필드로 독립적으로 관리된다** — 등록된 모든 책(소장 여부와 무관하게)의 읽기 상태(읽는 중/잠시 멈춤/완독)를 추적하며, 소장하지 않은 책(빌린 책·도서관 대출본)도 똑같이 산을 오른다. 소장 중 여부는 중복 구매 방지용 별도 체크박스일 뿐, 등반(읽기 진행률) 시각화와는 무관하게 동작한다.

앱 이름 **Read & Ridge**는 두 가지 의미를 담는다:
- **Read** — 독서 (소장 여부와 무관하게, 읽는 책들)
- **Ridge** — 산등성이 (읽기 진행률을 등반으로 표현하는 모티프)

## 기술 스택

| 영역 | 기술 |
|------|------|
| 프레임워크 | Next.js 14 (App Router) |
| 언어 | TypeScript |
| 스타일링 | Tailwind CSS |
| 백엔드/DB | Supabase (PostgreSQL + Auth) |
| 바코드 스캔 | @zxing/browser |
| 도서 검색 | 국립중앙도서관 서지정보(SEOJI) API, Open Library API |
| 아이콘 | lucide-react |

## 주요 기능

- **구글 소셜 로그인** (Supabase Auth)
- **책장 관리** — ISBN 바코드 스캔 또는 제목 검색으로 책 등록. "소장 중" 체크로 소유 여부 표시, 이미 소장 중인 책 재등록 시 "또 산 책이 됩니다!" 경고로 중복 구매 방지
- **열람 상황 추적** — 소장 여부와 무관하게 등록된 모든 책의 상태 관리 (읽는 중 / 완독 / 잠시 멈춤) + 페이지 진행률
- **월드맵** — 책장 전체가 픽셀 아트 산맥으로 시각화. 책의 두께 = 산 높이, 읽기 진행률 = 캐릭터 위치. 읽는 중인 책은 또렷한 전경 산으로, 미시작 책은 흐릿한 배경 설산으로 구분 표시
- **완등기록** — 완독한 책이 정상석으로 남는 기록. 별도 `hikes` 테이블이 아니라 `books`의 완독 데이터를 그대로 사용

## 페이지 구조

```
/login                → 구글 로그인
/dashboard            → 산책기록 (WorldMap + 읽는 중/잠시 멈춤 책 목록)
/dashboard/books      → /dashboard 로 리다이렉트 (구 링크/북마크 방어용)
/dashboard/hikes      → 완등기록 (WorldMap trophy 모드 + 완독 책 목록, 정상석)
/auth/callback        → OAuth 콜백
```

## 개발 서버

```bash
npm run dev   # http://localhost:3000 (또는 포트 변경 시 .env 확인)
```

환경변수 필요:
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `NL_API_KEY`
