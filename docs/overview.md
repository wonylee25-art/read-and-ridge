# Read & Ridge — 프로젝트 개요

## 컨셉

**출발점: "이 책, 내가 이미 샀나?"**

이 앱의 첫 번째 목적은 **내가 소유한 책을 파악하는 것**이다. 같은 책을 또 사는 일을 막기 위해 책장을 디지털로 기록해두는 데서 시작했다. ISBN 바코드 스캔으로 책을 빠르게 등록하고, 내 책장 전체를 한눈에 볼 수 있게 한다.

두 번째 목적은 **그 책장에서 열람 상황을 추적하는 것**이다. 소유 중인 책 중 어디까지 읽었는지, 읽다가 멈춘 건지, 완독했는지를 기록한다. 진행률은 픽셀 산의 등반으로 시각화된다.

즉, 순서는 이렇다:
1. **책장 기록** — 내가 가진 책을 모두 등록 (소유 확인이 주목적)
2. **열람 상황 추적** — 등록된 책의 독서 진행률 관리

앱 이름 **Read & Ridge**는 두 가지 의미를 담는다:
- **Read** — 독서 (소유하고, 읽는 책들)
- **Ridge** — 산등성이 (읽기 진행률을 등반으로 표현하는 모티프)

## 기술 스택

| 영역 | 기술 |
|------|------|
| 프레임워크 | Next.js 14 (App Router) |
| 언어 | TypeScript |
| 스타일링 | Tailwind CSS |
| 백엔드/DB | Supabase (PostgreSQL + Auth) |
| 바코드 스캔 | @zxing/browser |
| 도서 검색 | 카카오 도서 API, Open Library API |
| 아이콘 | lucide-react |

## 주요 기능

- **구글 소셜 로그인** (Supabase Auth)
- **책장 관리** — ISBN 바코드 스캔 또는 제목 검색으로 소유한 책 등록. 중복 구매 방지가 핵심 가치
- **열람 상황 추적** — 등록된 책의 상태 관리 (읽는 중 / 완독 / 잠시 멈춤) + 페이지 진행률
- **월드맵** — 책장 전체가 픽셀 아트 산맥으로 시각화. 책의 두께 = 산 높이, 읽기 진행률 = 캐릭터 위치
- **등산 기록** — 산 이름, 코스, 거리·고도·소요시간 기록

## 페이지 구조

```
/login                → 구글 로그인
/dashboard            → 홈 (WorldMap + 요약 통계)
/dashboard/books      → 독서 기록
/dashboard/hikes      → 등산 기록
/auth/callback        → OAuth 콜백
```

## 개발 서버

```bash
npm run dev   # http://localhost:3000 (또는 포트 변경 시 .env 확인)
```

환경변수 필요:
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `KAKAO_API_KEY`
