# Read & Ridge — 프로젝트 개요

## 컨셉

**출발점: "이 책, 내가 이미 샀나?"**

이 앱은 두 가지 목적을 가지며, 이 둘은 종속 관계가 아니라 같은 책 데이터 위에서 독립적으로 작동한다.

**소장 여부 관리** — 같은 책을 또 사는 일을 막기 위해 책을 등록할 때 "소장 중"을 체크한다. 이미 소장 중으로 등록된 책을 다시 소장으로 추가하려 하면 "또 산 책이 됩니다!" 팝업으로 경고한다. 빌린 책·도서관 대출본처럼 소장하지 않은 책은 체크하지 않으면 되고, 이 경우 중복 경고 대상에서 빠진다.

**열람 상황 추적** — 등록된 모든 책(소장 여부와 무관하게)의 읽기 상태를 기록한다. 어디까지 읽었는지, 읽다가 멈춘 건지, 완독했는지를 관리하며, 진행률은 픽셀 산의 등반으로 시각화된다.

즉, 책을 등록하는 순간 "소장 중" 여부와 "읽기 상태"는 서로 다른 두 필드로 따로 관리되고, 소장하지 않은 책도 똑같이 산을 오른다.

앱 이름 **Read & Ridge**는 두 가지 의미를 담는다:
- **Read** — 독서 (소장 여부와 무관하게, 읽는 책들)
- **Ridge** — 산등성이 (읽기 진행률을 등반으로 표현하는 모티프)

## 기술 스택

| 영역 | 기술 |
|------|------|
| 프레임워크 | Next.js 14 (App Router) |
| 언어 | TypeScript |
| 스타일링 | Tailwind CSS |
| 백엔드/DB | Supabase (PostgreSQL + Auth) |
| 바코드 스캔 | @zxing/browser |
| 도서 검색 | 국립중앙도서관 서지정보(SEOJI) API, Open Library API |
| 아이콘 | lucide-react |

## 주요 기능

- **구글 소셜 로그인** (Supabase Auth)
- **책장 관리** — ISBN 바코드 스캔 또는 제목 검색으로 책 등록. "소장 중" 체크로 소유 여부 표시, 이미 소장 중인 책 재등록 시 "또 산 책이 됩니다!" 경고로 중복 구매 방지
- **열람 상황 추적** — 소장 여부와 무관하게 등록된 모든 책의 상태 관리 (읽는 중 / 완독 / 잠시 멈춤) + 페이지 진행률
- **월드맵** — 책장 전체가 픽셀 아트 산맥으로 시각화. 책의 두께 = 산 높이, 읽기 진행률 = 캐릭터 위치
- **등산 기록** — 산 이름, 코스, 거리·고도·소요시간 기록

## 페이지 구조

```
/login                → 구글 로그인
/dashboard            → 홈 (WorldMap + 요약 통계)
/dashboard/books      → 독서 기록
/dashboard/hikes      → 등산 기록
/auth/callback        → OAuth 콜백
```

## 개발 서버

```bash
npm run dev   # http://localhost:3000 (또는 포트 변경 시 .env 확인)
```

환경변수 필요:
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `NL_API_KEY`
