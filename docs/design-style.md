# 디자인 스타일 가이드

> 문서 출처: PRD v1.1 (2026.06.26) + 레퍼런스 이미지 기준
> 마지막 업데이트: 2026.06.29 (Canvas 구현 반영, SVG 잔재 정리)

---

## 렌더링 현황

`WorldMap.tsx`는 **HTML5 Canvas 기반으로 재구현 완료**됐다(2026.06.29). `ctx.getContext('2d')` + `imageSmoothingEnabled = false` + `requestAnimationFrame` 게임 루프로 동작하며, 고채도 산 색상·픽셀 구름 오브젝트·레이어드 실루엣이 적용돼 있다. 과거 SVG `<rect>` 방식은 더 이상 사용하지 않는다.

---

## 핵심 무드

**부드러운 파스텔 픽셀 풍경 × 대자연 파노라마 × 차분한 기록 감성**

쨍한 원색의 아케이드 게임이 아니라, **밝고 부드러운 파스텔 톤의 픽셀 풍경화**다. 하늘은 맑고 환한 주간 톤(밝은 하늘색 → 연한 하늘색)을 기본 인상으로 하고, 흰 픽셀 구름과 따뜻한 태양이 떠 있다. 산은 각 KDC 테마별로 채도 낮고 명도 높은 파스텔 색(연분홍·황금브라운·청록·회색)으로 은은하게 구분되며, 능선 대비는 부드럽게 둔다. 침엽수·벚꽃잎·작은 캐릭터·정상석 같은 자연 오브젝트가 풍경을 채운다.

기조: 차분함 · 친근함 · 담백함. 시끄러운 게임 월드가 아니라 **"조용히 쌓여가는 내 독서 풍경"**처럼 느껴져야 한다. 강렬한 대비나 드라마틱한 연출보다, 오래 봐도 편안한 톤을 우선한다.

## 픽셀 아트 시스템

### 렌더링 엔진 — HTML5 Canvas
- Canvas `fillRect()` 드로잉 (구현 완료)
- `ctx.imageSmoothingEnabled = false`로 픽셀 경계 선명 유지
- 애니메이션: `requestAnimationFrame` 기반 게임 루프
- 산, 캐릭터, 구름, 파티클 모두 Canvas 단일 레이어에서 처리

### 산 (Mountain)
- 1 픽셀 단위 = 10px Canvas 블록
- 삼각형 피라미드 구조: 행마다 한 칸씩 넓어지는 계단식 계층
- 구성 영역: 정상(snow) → 능선(edge) → 산 몸체(fill) → 지면(base)
- **레퍼런스 이미지 기준**: 산들이 서로 다른 색상으로 뚜렷하게 구분됨. 같은 화면에 보라/초록/회색 산이 나란히 존재

#### 입체 명암 (3D shading)
산을 한 색으로 평칠하지 않고, **빛이 왼쪽 위에서 비치는** 것으로 가정해 면을 나눠 입체감을 준다.

- **왼쪽 면(밝은 면)**: `fill`보다 한 톤 밝게 (정상 highlight 쪽)
- **오른쪽 면(그림자 면)**: `fill`보다 한 톤 어둡게 (= `edge` 계열 활용)
- **정상**: 다이아몬드형 `snow` 블록으로 눈쌓임
- **중앙 능선**: 좌/우 면 경계에 한 칸짜리 `edge` 라인을 세로로 그어 봉우리 선을 또렷하게
- 도트는 잘게 (작은 블록)일수록 명암 경계가 부드럽게 읽힌다.

#### 산 모양 다양화 (silhouette variation)
모든 산이 똑같은 좌우대칭 삼각형이면 단조롭다. **책마다 다른 실루엣**을 주되, 분류(KDC)는 색으로만 표현한다.

- **색톤 = KDC 테마(고정)**: 같은 분야의 책이 여러 권이면 **모두 같은 테마 색톤**을 공유한다. (예: 과학/기술 책 3권 → 모두 nature 청록 계열)
- **실루엣 = 책 id 시드(가변)**: 같은 분야여도 책마다 봉우리 형태가 달라야 한다. 책 `id`(또는 제목) 해시를 시드로 실루엣 종류를 결정 → **같은 책은 항상 같은 모양**, 책장 전체는 다채로운 산맥.
- **높이 = 페이지 수(의미)**: 기존 `level 1~4`(`STEPS_BY_LEVEL`)로 산 높이를 정하는 규칙은 유지. 두꺼운 책일수록 높은 산.

실루엣 종류(시드로 선택):

| 종류 | 형태 | 비고 |
|------|------|------|
| 뾰족(sharp) | 가파른 대칭 삼각 | 기본형 |
| 완만(broad) | 낮고 넓은 삼각 | |
| 비대칭(skew) | 정점이 좌/우로 치우침 | 시드로 방향 결정 |
| 쌍봉(twin) | 봉우리 2개 + 능선 안부 | snow 블록 2개 |
| 고원(plateau) | 정상이 평평 | snow 가로 블록 |

> 정리 원칙: **"같은 분야 = 비슷한 컬러톤, 다른 실루엣"**. 색은 분류를, 형태는 개별 책을 구분한다.
>
> ⚠️ 구현 시 주의: 실루엣을 바꾸면 형태 계산(`getMountainGeometry`)과 산 클릭 hit-test(`getMountainRects`)의 폭/높이 로직을 함께 손봐야 한다. (현재 코드는 대칭 삼각형 단일 형태 기준 — 본 항목은 **목업·가이드 단계의 목표**이며 아직 코드 미반영.)

목업 참고 이미지: `worldmap-mockup-pastel.png`(전체 풍경), `mountain-shape-variations.png`(같은 색톤·다른 실루엣).

### 구름 (Clouds)
- 흰 픽셀 블록 조합의 구름 오브젝트 (구현 완료)
- 주간(06~16시)에만 노출
- 천천히 좌측 방향으로 흘러가는 애니메이션

### 캐릭터
- **6×8 컬러 도트 사람** — 머리·얼굴·옷·다리가 구분되는 사람 형태. 단색 실루엣이 아니라 색으로 부위를 나눈다.
- 픽셀 크기: 3px/dot (`CPX = 3`)
- 픽셀맵: 행 0 머리카락 → 1 얼굴 → 2 어깨 → 3 양팔 벌린 몸통 → 4~5 몸통/골반 → 6~7 두 다리. 색 코드 `H`=머리카락, `S`=피부, `B`=옷, `L`=다리.

  ```
  서있기(A)        걷기(B)
  .HHHH.           .HHHH.   ← 머리카락
  .SSSS.           .SSSS.   ← 얼굴(피부)
  ..BB..           ..BB..   ← 어깨
  BBBBBB           BBBBBB   ← 양팔 벌린 몸통
  .BBBB.           .BBBB.   ← 몸통
  .BBBB.           .BBBB.   ← 골반
  .L..L.           LL..LL   ← 다리 (걷기 시 벌림)
  .L..L.           L....L
  ```

- 색상(`CHAR_COLORS` — 코드가 진실): 머리카락 `#5a3a22`, 피부 `#f1c9a5`, 옷 `#4a7fc0`, 다리 `#3a3a3a`. 옷/머리색은 이 객체 한 곳에서 관리한다.
- 드로잉(`drawChar`)은 픽셀맵 크기를 행 배열에서 자동 계산하므로, 픽셀맵을 키워도 발 위치(능선 위 정렬)가 유지된다.
- 위치: 읽는 중인 산의 능선 위. 진행률이 높을수록 정상에 가깝게 올라간다. `status === 'reading'`인 책에만 표시.

### 정상석 (Summit Stone)
- 완독 시 정상에 솟아오르는 픽셀 바위 오브젝트
- 표면 각인: 책 제목 + 완독일(YYYY.MM.DD) — 픽셀 서체, 극도의 미니멀
- 색상: 해당 도서의 KDC 산맥 컬러 상속

## 색상 시스템

### KDC 기반 산 색상 테마 (4종)

> KDC 값은 유저에게 텍스트로 절대 노출하지 않는다. 색상으로만 암묵적으로 표현.

> 정확한 hex는 코드(`components/worldmap/WorldMap.tsx`의 `KDC_THEME`)가 진실 공급원이다. 아래 표는 그 값과 1:1로 동기화한다 — 코드를 바꾸면 이 표도 함께 갱신할 것.

#### KDC 테마별 색상값 (코드 `KDC_THEME` 기준)

각 산은 `fill`(몸체) · `edge`(능선/베이스) · `snow`(정상) · `base`(미사용 예비/짙은 톤) 4색으로 구성된다.

| 테마 | KDC | fill (몸체) | edge (능선) | snow (정상) | base | 추가 오브젝트 |
|------|-----|------|------|------|------|------------|
| mystery — 총류/철학/종교 (청회) | 0, 1, 2 | `#5b8dd9` | `#2c4d8a` | `#e8f0ff` | `#1e3560` | 주변부 안개 파티클 |
| earth — 사회/예술/역사 (황토/브라운) | 3, 7, 9 | `#c97b2e` | `#7a4010` | `#fff0cc` | `#5a2e08` | 바위 절벽 텍스처 블록 |
| nature — 과학/기술 (에메랄드) | 4, 5 | `#2db86a` | `#166b3a` | `#d4ffea` | `#0d4022` | 침엽수 픽셀 오브젝트 |
| fantasy — 어학/문학 (라벤더/보라) | 6, 8 | `#9b59d0` | `#5a1f8a` | `#f0e0ff` | `#3b1060` | 흩날리는 꽃잎 파티클 |
| default — KDC 없음 (그린) | — | `#3aac6e` | `#1a6640` | `#d6f5e6` | `#0e4228` | — |

**KDC → 테마 매핑**(`getTheme`): 첫 자리 `0,1,2`→mystery, `3,7,9`→earth, `4,5`→nature, `6,8`→fantasy. KDC 미상이면 인덱스 순환으로 배정.

## 디자인 토큰

코드 진실(source of truth)은 `globals.css`의 CSS 변수와 `tailwind.config.ts`이다. 컴포넌트에 raw 값(`#171717` 등)을 직접 박지 말고 아래 토큰을 경유한다.

### 색상 토큰 (현재 정의됨)

| 토큰 | 라이트 | 다크 | 용도 |
|------|--------|------|------|
| `--background` | `#ffffff` | `#0a0a0a` | 페이지 배경 (Tailwind `bg-background`) |
| `--foreground` | `#171717` | `#ededed` | 기본 텍스트 (Tailwind `text-foreground`) |

> 월드맵 Canvas 내부 색상(하늘/산/캐릭터)은 이 토큰 체계와 **분리**된다. Canvas는 직접 hex를 쓰며, 그 값들은 위 「색상 시스템」 표가 진실이다.

### 간격 / 라운드 / 그림자 (권장 토큰화)

현재 컴포넌트는 Tailwind 기본 스케일을 직접 쓰고 있다. 일관성을 위해 다음 규칙으로 좁혀 쓴다.

| 카테고리 | 허용 값 | 비고 |
|----------|---------|------|
| radius (카드) | `rounded-2xl` | 보조 페이지 카드 |
| radius (버튼/인풋) | `rounded-xl` / `rounded-lg` | |
| radius (뱃지) | `rounded-full` | |
| shadow | `shadow-sm` | 강한 그림자 금지 (픽셀 무드 유지) |
| spacing | `2 / 3 / 4 / 6` (8pt 계열) | 임의 값 `px-[13px]` 지양 |

### 폰트 토큰 (실제 정의됨)

| 변수 | 폰트 | 용도 |
|------|------|------|
| `--font-geist-sans` | Geist Sans (`localFont`) | UI 본문 |
| `--font-geist-mono` | Geist Mono (`localFont`) | 코드/수치, 픽셀 레이블 대체 |

> 기존 가이드의 "픽셀 월드맵 내부 `fontFamily: monospace`"는 Canvas 텍스트에만 적용. DOM UI는 Geist Sans를 기본으로 한다.

## 반응형 / 브레이크포인트

월드맵(`/`)이 메인이며 **모바일 우선**으로 설계한다. Tailwind 기본 브레이크포인트를 사용한다.

| 구간 | 폭 | 월드맵 거동 | UI 거동 |
|------|-----|------------|---------|
| 모바일 | `< 640px` (기본) | Canvas 가로 스크롤, 풀블리드 | 컨트롤은 하단 오버레이 패널 |
| 태블릿 | `sm 640px ~ md 768px` | Canvas 폭에 맞춰 스케일 | 패널 폭 확장 |
| 데스크탑 | `lg 1024px+` | 파노라마 와이드, 산 다수 표시 | 보조 페이지(books/hikes)는 카드 그리드 |

원칙: 클래스는 모바일 기본값 먼저 쓰고 `sm:`/`md:`/`lg:`로 키운다. Canvas는 컨테이너 폭 기준으로 리사이즈하되 `imageRendering: pixelated`를 유지해 확대 시에도 픽셀 경계를 보존한다.

## 상태별 UI (로딩 / 에러 / 빈 상태)

데이터 페칭(도서 검색, Supabase 조회)이 있는 모든 화면은 4가지 상태를 명시적으로 디자인한다.

| 상태 | 표현 | 비고 |
|------|------|------|
| 로딩 | 픽셀 스피너 또는 스켈레톤 (`animate-pulse` 회색 블록) | 스피너는 픽셀 무드 유지 |
| 빈 상태 | 데모 산맥 씬 + "첫 책을 등록해 첫 봉우리를 세워보세요" 안내 | WorldMap의 `books.length === 0` 데모 씬 재활용 |
| 에러 | 픽셀 말풍선 + 재시도 버튼 | 토스트보다 인라인 우선 |
| 성공/완료 | 8비트 CLEAR! 세레모니 (완등 시) | 별도 섹션 참고 |

빈 상태를 "아무것도 없는 검은 화면"으로 두지 않는다 — 항상 데모 씬이나 안내를 보여준다.

## 모션 / 접근성 (a11y)

### prefers-reduced-motion

OS에서 "동작 줄이기"를 켠 사용자를 위해 비필수 애니메이션을 정지한다.

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.001ms !important;
    transition-duration: 0.001ms !important;
  }
}
```

Canvas의 `requestAnimationFrame` 루프도 이 설정을 감지하면 캐릭터 바운스·모닥불·구름 흐름을 정지 프레임으로 고정한다(하늘/산 정적 렌더는 유지).

### 색상 대비

KDC 색상은 "의미"를 색으로만 전달하므로(텍스트 비노출 원칙), **색맹 사용자**를 위해 산 실루엣 형태·정상석 유무·캐릭터 위치 등 **색 이외의 단서**를 항상 병행한다. 색만으로 정보를 구분하지 않는다.

### 시맨틱 / 스크린리더

- Canvas는 이미 `aria-label="Read & Ridge 픽셀 독서 산맥 월드맵"` 보유. 도서 데이터가 바뀌면 `aria-label`도 갱신(예: "○○ 외 3권의 산맥")한다.
- 색·픽셀로만 표현되는 진행률은 보조 페이지(`/dashboard/books`)에서 숫자/진행바로 접근 가능해야 한다(Canvas의 백업 경로).
- 인터랙티브 요소(산 클릭 모달 등)는 키보드 포커스·`Esc` 닫기를 지원한다.

### 포커스 표시

`focus:ring-2 focus:ring-gray-900` (기존 인풋 원칙) 외에, 다크 배경 위 요소는 `focus:ring-offset` 또는 밝은 링 색을 써서 포커스가 보이게 한다.

## 다크모드 정책

현재 `globals.css`는 `prefers-color-scheme`로 OS 설정을 따른다(`--background`/`--foreground` 자동 전환).

- 보조 페이지(books/hikes 카드 UI)는 OS 다크모드를 따른다.
- 월드맵(`/`)은 **시간대 하늘 색상이 사실상 자체 다크모드** 역할을 하므로, OS 다크모드와 무관하게 시간 기반 배경을 우선한다.
- 라이트/다크 양쪽에서 텍스트 대비(WCAG AA, 본문 4.5:1)를 만족하는지 확인한다.

## 로케일 / 카피 톤

- 페이지 언어: `<html lang="ko">` (확정).
- UI 카피는 한국어, 간결·담백한 기록 감성. 과장된 게임 문구는 세레모니(CLEAR!)에만 한정.
- 날짜 표기: `YYYY.MM.DD` (정상석 각인과 통일).
