# 디자인 스타일 가이드

> 문서 출처: PRD v1.1 (2026.06.26) + 레퍼런스 이미지 기준
> 마지막 업데이트: 2026.06.29 (Canvas 구현 반영, SVG 잔재 정리)

---

## 렌더링 현황

`WorldMap.tsx`는 **HTML5 Canvas 기반으로 재구현 완료**됐다(2026.06.29). `ctx.getContext('2d')` + `imageSmoothingEnabled = false` + `requestAnimationFrame` 게임 루프로 동작하며, 고채도 산 색상·픽셀 구름 오브젝트·레이어드 실루엣이 적용돼 있다. 과거 SVG `<rect>` 방식은 더 이상 사용하지 않는다.

---

## 핵심 무드

**16비트 아케이드 픽셀 아트 × 대자연 파노라마 × 기록 감성**

슈퍼 패미컴 시대 게임 배경화면처럼 선명하고 생동감 있는 컬러. 하늘은 쨍하게 파랗고, 산은 각 KDC 테마별로 뚜렷하게 다른 원색 계열로 채색된다. 조용한 기록앱이 아니라 "내 독서 역사의 게임 월드"처럼 느껴져야 한다.

## 픽셀 아트 시스템

### 렌더링 엔진 — HTML5 Canvas
- Canvas `fillRect()` 드로잉 (구현 완료)
- `ctx.imageSmoothingEnabled = false`로 픽셀 경계 선명 유지
- 애니메이션: `requestAnimationFrame` 기반 게임 루프
- 산, 캐릭터, 구름, 파티클 모두 Canvas 단일 레이어에서 처리

### 산 (Mountain)
- 1 픽셀 단위 = 10px Canvas 블록
- 삼각형 피라미드 구조: 행마다 한 칸씩 넓어지는 계단식 계층
- 구성 영역: 정상(snow) → 능선(edge) → 산 몸체(fill) → 지면(base)
- **레퍼런스 이미지 기준**: 산들이 서로 다른 색상으로 뚜렷하게 구분됨. 같은 화면에 보라/초록/회색 산이 나란히 존재

### 구름 (Clouds)
- 흰 픽셀 블록 조합의 구름 오브젝트 (구현 완료)
- 주간(06~16시)에만 노출
- 천천히 좌측 방향으로 흘러가는 애니메이션

### 캐릭터
- 5×5 픽셀 아트 도트 인형 (`CHAR_ROWS = ['.XXX.','XXXXX','XX.XX','.X.X.','.X.X.']`)
- 픽셀 크기: 3px/dot
- 색상: #1a1a1a (딥 차콜, 배경 하늘과 대비)

### 정상석 (Summit Stone)
- 완독 시 정상에 솟아오르는 픽셀 바위 오브젝트
- 표면 각인: 책 제목 + 완독일(YYYY.MM.DD) — 픽셀 서체, 극도의 미니멀
- 색상: 해당 도서의 KDC 산맥 컬러 상속

## 색상 시스템

### KDC 기반 산 색상 테마 (4종)

> KDC 값은 유저에게 텍스트로 절대 노출하지 않는다. 색상으로만 암묵적으로 표현.

> 과거 SVG 저채도 색상값은 폐기됐다. 현재 Canvas 구현은 아래 고채도 방향을 따른다. 정확한 hex는 코드(`components/worldmap/WorldMap.tsx`)가 진실이다.

#### 색상 방향 (고채도 — 레퍼런스 이미지 기준)

레퍼런스 이미지에서 산들은 **선명하고 대비가 강한 원색 계열**을 사용한다.

| 테마명 | KDC | 방향 | 추가 오브젝트 |
|--------|-----|------|------------|
| mystery | 0, 1, 2 (총류/철학/종교) | 밝은 회백색 + 짙은 청회색 능선 | 주변부 안개 파티클 |
| earth | 3, 7, 9 (사회/예술/역사) | 진한 황토/브라운 + 적갈색 능선 | 바위 절벽 텍스처 블록 |
| nature | 4, 5 (과학/기술) | 선명한 에메랄드 그린 + 짙은 초록 능선 | 침엽수 픽셀 오브젝트 |
| fantasy | 6, 8 (어학/문학) | 채도 높은 라벤더 + 보라 능선 | 흩날리는 꽃잎 파티클 |

→ 색상값은 Canvas 구현에 반영 완료. 조정 시 코드와 이 표를 함께 갱신할 것.

|--------|-----|--------|---------|
| 새벽 00~06시 | #060b22 | #0d1540 | 별빛 60개, 랜덤 배치 |
| 주간 06~16시 | #2980b9 | #87ceeb | 태양 (노란 원, #ffe066) |
| 황혼 16~19시 | #d04010 | #f08050 | 태양 (주황 원, #ff9944) |
| 야간 19~24시 | #0e1248 | #141840 | 별빛 60개 |

### 지면 (Ground Strip)
```
linear-gradient(to bottom, #2a4828, #182818)
```
높이 52px, 화면 하단 고정.

### 하단 UI 패널 (월드맵 아래 컨트롤 영역)
월드맵(`/`)이 메인이므로 별도 대시보드 UI보다 **오버레이/패널 형태**가 목표.
- 배경: 반투명 다크 (`rgba(0,0,0,0.6)` 계열) 또는 픽셀 아트 UI 박스
- 버튼/인풋: 픽셀 테두리 스타일 지향
- 현재 `/dashboard` 체계의 화이트 카드 UI는 **보조 페이지**(books, hikes 목록)에만 유지

## 디자인 토큰

코드 진실(source of truth)은 `globals.css`의 CSS 변수와 `tailwind.config.ts`이다. 컴포넌트에 raw 값(`#171717` 등)을 직접 박지 말고 아래 토큰을 경유한다.

### 색상 토큰 (현재 정의됨)

| 토큰 | 라이트 | 다크 | 용도 |
|------|--------|------|------|
| `--background` | `#ffffff` | `#0a0a0a` | 페이지 배경 (Tailwind `bg-background`) |
| `--foreground` | `#171717` | `#ededed` | 기본 텍스트 (Tailwind `text-foreground`) |

> 월드맵 Canvas 내부 색상(하늘/산/캐릭터)은 이 토큰 체계와 **분리**된다. Canvas는 직접 hex를 쓰며, 그 값들은 위 「색상 시스템」 표가 진실이다.

### 간격 / 라운드 / 그림자 (권장 토큰화)

현재 컴포넌트는 Tailwind 기본 스케일을 직접 쓰고 있다. 일관성을 위해 다음 규칙으로 좁혀 쓴다.

| 카테고리 | 허용 값 | 비고 |
|----------|---------|------|
| radius (카드) | `rounded-2xl` | 보조 페이지 카드 |
| radius (버튼/인풋) | `rounded-xl` / `rounded-lg` | |
| radius (뱃지) | `rounded-full` | |
| shadow | `shadow-sm` | 강한 그림자 금지 (픽셀 무드 유지) |
| spacing | `2 / 3 / 4 / 6` (8pt 계열) | 임의 값 `px-[13px]` 지양 |

### 폰트 토큰 (실제 정의됨)

| 변수 | 폰트 | 용도 |
|------|------|------|
| `--font-geist-sans` | Geist Sans (`localFont`) | UI 본문 |
| `--font-geist-mono` | Geist Mono (`localFont`) | 코드/수치, 픽셀 레이블 대체 |

> 기존 가이드의 "픽셀 월드맵 내부 `fontFamily: monospace`"는 Canvas 텍스트에만 적용. DOM UI는 Geist Sans를 기본으로 한다.

## 반응형 / 브레이크포인트

월드맵(`/`)이 메인이며 **모바일 우선**으로 설계한다. Tailwind 기본 브레이크포인트를 사용한다.

| 구간 | 폭 | 월드맵 거동 | UI 거동 |
|------|-----|------------|---------|
| 모바일 | `< 640px` (기본) | Canvas 가로 스크롤, 풀블리드 | 컨트롤은 하단 오버레이 패널 |
| 태블릿 | `sm 640px ~ md 768px` | Canvas 폭에 맞춰 스케일 | 패널 폭 확장 |
| 데스크탑 | `lg 1024px+` | 파노라마 와이드, 산 다수 표시 | 보조 페이지(books/hikes)는 카드 그리드 |

원칙: 클래스는 모바일 기본값 먼저 쓰고 `sm:`/`md:`/`lg:`로 키운다. Canvas는 컨테이너 폭 기준으로 리사이즈하되 `imageRendering: pixelated`를 유지해 확대 시에도 픽셀 경계를 보존한다.

## 상태별 UI (로딩 / 에러 / 빈 상태)

데이터 페칭(도서 검색, Supabase 조회)이 있는 모든 화면은 4가지 상태를 명시적으로 디자인한다.

| 상태 | 표현 | 비고 |
|------|------|------|
| 로딩 | 픽셀 스피너 또는 스켈레톤 (`animate-pulse` 회색 블록) | 스피너는 픽셀 무드 유지 |
| 빈 상태 | 데모 산맥 씬 + "첫 책을 등록해 첫 봉우리를 세워보세요" 안내 | WorldMap의 `books.length === 0` 데모 씬 재활용 |
| 에러 | 픽셀 말풍선 + 재시도 버튼 | 토스트보다 인라인 우선 |
| 성공/완료 | 8비트 CLEAR! 세레모니 (완등 시) | 별도 섹션 참고 |

빈 상태를 "아무것도 없는 검은 화면"으로 두지 않는다 — 항상 데모 씬이나 안내를 보여준다.

## 모션 / 접근성 (a11y)

### prefers-reduced-motion

OS에서 "동작 줄이기"를 켠 사용자를 위해 비필수 애니메이션을 정지한다.

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.001ms !important;
    transition-duration: 0.001ms !important;
  }
}
```

Canvas의 `requestAnimationFrame` 루프도 이 설정을 감지하면 캐릭터 바운스·모닥불·구름 흐름을 정지 프레임으로 고정한다(하늘/산 정적 렌더는 유지).

### 색상 대비

KDC 색상은 "의미"를 색으로만 전달하므로(텍스트 비노출 원칙), **색맹 사용자**를 위해 산 실루엣 형태·정상석 유무·캐릭터 위치 등 **색 이외의 단서**를 항상 병행한다. 색만으로 정보를 구분하지 않는다.

### 시맨틱 / 스크린리더

- Canvas는 이미 `aria-label="Read & Ridge 픽셀 독서 산맥 월드맵"` 보유. 도서 데이터가 바뀌면 `aria-label`도 갱신(예: "○○ 외 3권의 산맥")한다.
- 색·픽셀로만 표현되는 진행률은 보조 페이지(`/dashboard/books`)에서 숫자/진행바로 접근 가능해야 한다(Canvas의 백업 경로).
- 인터랙티브 요소(산 클릭 모달 등)는 키보드 포커스·`Esc` 닫기를 지원한다.

### 포커스 표시

`focus:ring-2 focus:ring-gray-900` (기존 인풋 원칙) 외에, 다크 배경 위 요소는 `focus:ring-offset` 또는 밝은 링 색을 써서 포커스가 보이게 한다.

## 다크모드 정책

현재 `globals.css`는 `prefers-color-scheme`로 OS 설정을 따른다(`--background`/`--foreground` 자동 전환).

- 보조 페이지(books/hikes 카드 UI)는 OS 다크모드를 따른다.
- 월드맵(`/`)은 **시간대 하늘 색상이 사실상 자체 다크모드** 역할을 하므로, OS 다크모드와 무관하게 시간 기반 배경을 우선한다.
- 라이트/다크 양쪽에서 텍스트 대비(WCAG AA, 본문 4.5:1)를 만족하는지 확인한다.
