## 요약

전반적으로 **핵심 공간 설계 원칙은 의도대로 구현돼 있다.** 가로축 타임라인, 세로축 고도, Z축 레이어, 시간대 하늘, KDC 채색, 캐릭터/모닥불 애니메이션 모두 동작한다. 다만 ① 문서가 옛 값으로 남아 코드와 어긋난 곳, ② 기획 의도와 구현 수치가 미세하게 다른 곳, ③ 코드 내부 중복으로 정합성 위험이 있는 곳이 있다.

판정 범례: ✅ 일치 · ⚠️ 경미한 차이(판단 필요) · 📄 문서만 낡음 · 🔧 리팩토링 대상

## 1. 공간 설계 원칙 (product-direction.md)

| 의도 | 구현 | 판정 |
|------|------|------|
| 가로축 = 등록 시간순 고정 (왼=과거, 오른=현재) | `page.tsx`에서 `.order('created_at', { ascending: true })` | ✅ |
| 유저가 순서 변경 불가 | 정렬이 서버 쿼리에 고정, 재배열 UI 없음 | ✅ |
| 세로축 = 총 페이지 → 산 높이(1~4단계) | `getLevel(pages)` → `STEPS_BY_LEVEL` | ✅ |
| 세로축 = 읽은 페이지 → 캐릭터 위치 | `charRow = round((1 - progress) * (steps - 1))` | ✅ |
| Z축 전경(읽는 중): 불투명100%, 크기100%, 드롭섀도 | `opacity 1.0 / scale 1.0` + 드롭섀도 글로우 | ✅ |
| Z축 원경(미시작/완독): **투명도 60%, 0.8배** | 코드는 **opacity 0.5 / scale 0.82** | ⚠️ 수치 차이 |

**⚠️ 차이 1 — 원경 레이어 수치**: 기획은 "60% 투명도 + 0.8배", 코드는 "50% 투명도(`opacity 0.5`) + 0.82배(`scale 0.82`)". 시각적으로 큰 차이는 아니나, 기획값과 다르다. design-style.md의 Z레이어 표(50%/0.82)와는 일치 → 문서 내부에서도 product-direction과 design-style이 서로 다른 값을 적고 있음. **하나로 통일 필요.**

## 4. 산 레벨 / 페이지 기준

| 의도 | 구현 | 판정 |
|------|------|------|
| 200p 미만 → L1(5행) | `pages < 200 → 1` | ✅ |
| 200~399 → L2(7행) | `pages < 400 → 2` | ✅ |
| 400~599 → L3(9행) | `pages < 600 → 3` | ✅ |
| 600+ → L4(11행) | `else → 4` | ✅ |
| 페이지 **누락 시 300쪽 기본값**(2단계) | `getLevel(null) → 2` (행 수만 2단계, 실제 분모는 null 처리) | ⚠️ |

**⚠️ 차이 4 — 페이지 누락 처리**: 기획(product-direction)은 "총 쪽수 누락 시 **300쪽 기본값 배정** + 유저 수정 입력박스"를 명시. 코드는 산 높이 레벨만 2단계로 처리하고, 진행률 계산에서는 `total_pages`가 `null`이면 progress를 0으로 둔다(`book.total_pages ? ... : 0`). 즉 "300쪽 기본값"이 데이터로 주입되지는 않음. 등록 단계에서 300 기본값을 넣는 로직이 빠져 있어 **누락 도서는 영원히 0% 캐릭터**가 된다. → 등록 파이프라인 보완 필요(미구현).

## 5. 도서 등록 / 책장 (overview.md, features/books.md)

| 의도 | 구현 | 판정 |
|------|------|------|
| ISBN 바코드 스캔 등록 | `BarcodeScanner.tsx` (@zxing) | ✅ |
| 제목 검색 등록 (국립중앙도서관 API) | `AddBookForm` + `/api/books/search?q=` | ✅ |
| 중복 ISBN 차단 + Shake + 말풍선 | **미구현** | ❌ 미구현 |
| KDC 산 색상 자동 배정 | 검색 응답에서 KDC/SUBJECT 추출해 저장(2026.07.12), 없으면 book.id 해시 순환 폴백 | ✅ |
| 상태 관리 reading/completed/paused | `BookCard` 드롭다운 | ✅ |
| 완독 자동 처리 | `current_page >= total_pages` 자동 completed | ✅ |

**❌ 미구현 5 — 중복 ISBN 방어**: overview.md가 "중복 구매 방지가 **핵심 가치**"라고 강조하는데, 정작 같은 ISBN 재등록을 막는 로직(Shake + `[이미 등산로가 개척된 산입니다!]` 말풍선)이 없다. **제품 핵심 가치라 우선순위 높음.**

**✅ 해소 6 — KDC 채색의 근본 한계 (2026.07.12 해결)**: `fetchNL`이 SEOJI 응답의 `KDC`·`SUBJECT` 필드를 추출해 `addBook`까지 전달하도록 보완했다(`app/api/books/search/route.ts`의 `resolveKdc`). 실제 API 응답을 확인해보니 `KDC`(정본)는 CIP 사전등록 도서에서만 채워지고 대부분의 최신/전자책/자가출판 도서는 빈 문자열이었음 — `SUBJECT`(KDC 대분류 한 자리 숫자, 예: 문학 `"8"`)가 훨씬 넓게 채워져 있어 이를 1차 폴백으로 함께 사용하도록 구현. 둘 다 없는 책은 기존과 동일하게 book.id 해시 순환으로 색이 배정된다(동작 변경 없음). 남은 한계: `SUBJECT`는 근사값이라 정본 KDC와 항상 일치하지는 않을 수 있음.
