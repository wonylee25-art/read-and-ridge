## 저장 위치

별도 `profiles` 테이블 없이 `auth.users`의 `user_metadata.nickname`에 저장한다
(`supabase.auth.updateUser({ data: { nickname } })`). 다른 유저에게 보여줄 일이
없는 "본인 전용" 값(인사말, 캡처 워터마크)이라 RLS·마이그레이션이 필요 없는 이
방식으로 충분하다고 판단함.

> 나중에 다른 유저의 닉네임도 보여주는 소셜 기능(친구 비교, 리더보드 등)이 생기면
> 그때 `profiles` 테이블로 옮기는 게 맞다 — 지금은 과설계하지 않음.

## 기본값 (`app/dashboard/page.tsx`)

우선순위: `user_metadata.nickname`(직접 설정) → `user_metadata.full_name`(구글
프로필 이름, OAuth 시 자동으로 채워짐) → `user_metadata.name` → `'산책자'`(최후
기본값). 대부분의 유저는 별도 설정 없이도 구글 이름으로 바로 인사말이 뜬다.

## UI

`components/dashboard/Greeting.tsx` — `WorldMapClient.tsx`에서 `AddBookBar`
바로 위, 책 추가 버튼과 같은 오른쪽 정렬로 렌더링. `authenticated && nickname`일
때만 표시(완등기록 페이지에는 `nickname` prop을 안 넘겨서 자연스럽게 숨겨짐).

편집은 `BookCard.tsx`의 제목/저자 인라인 수정과 동일한 패턴 — 연필 아이콘 클릭 →
입력창 + 저장/취소. 별도 모달 없음.

## 인사말 문구 (시간대별) — 2026.07.12 갱신

> ⚠️ 위 배너에서 언급한 대로 UI가 `ProfileTrigger.tsx`(산책자 증표 트리거)로
> 옮겨가면서 문구 체계도 4구간 고정 문구에서 아래 12구간·구간당 2개 후보로
> 바뀌었다. 이전 버전(새벽/주간/황혼/야간 4구간)은 폐기됨.

`components/dashboard/ProfileTrigger.tsx`의 `GREETING_PHRASES` 배열에 하드코딩돼
있다(DB 저장 아님). 2시간 단위 12구간으로 나누고, 구간마다 후보 문구 2개 중
하나를 `pickGreetingPhrase()`가 매번 랜덤(50%)으로 골라 "{닉네임}님, {문구}"
형태로 보여준다. 배열 인덱스는 `Math.floor(hour / 2)`.

| 시간대 | 문구 후보 1 | 문구 후보 2 |
|--------|------------|------------|
| 00:00 ~ 02:00 | 위로가 필요하신가요? | 밤이 깊어가네요 |
| 02:00 ~ 04:00 | 고요한 새벽이네요 | 책장 넘기는 소리가 들리네요 |
| 04:00 ~ 06:00 | 이른 시작이네요 | 첫 페이지를 열어볼까요? |
| 06:00 ~ 08:00 | 기분 좋은 아침이에요 | 아침 첫 문장을 만나볼까요? |
| 08:00 ~ 10:00 | 새소리가 들리나요? | 가볍게 출발해 볼까요? |
| 10:00 ~ 12:00 | 하늘을 올려다봐요 | 싱그러운 바람을 즐겨요 |
| 12:00 ~ 14:00 | 잠시 숨 고를 시간이에요 | 잠시 책갈피를 꽂아둘까요? |
| 14:00 ~ 16:00 | 나른한 시간이죠? | 달콤한 충전이 필요해요 |
| 16:00 ~ 18:00 | 슬슬 기지개를 켜볼까요? | 마음에 닿은 문장이 있나요? |
| 18:00 ~ 20:00 | 노을이 참 예쁘네요 | 오늘 하늘 보셨나요? |
| 20:00 ~ 22:00 | 오늘도 고생 많았어요 | 온전히 나만의 시간을 보내볼까요 |
| 22:00 ~ 24:00 | 밤바람이 선선하죠? | 오래 남을 문장을 만났나요? |

브라우저 로컬 시각 기준, 페이지 마운트 시점에 1회만 계산(실시간 갱신 타이머 없음
— WorldMap 하늘과 달리 텍스트라 분 단위 정확도가 중요하지 않음). 서버 렌더 시점엔
기본값 "반가워요"가 잠깐 보이다가 `useEffect`에서 위 표 기준 문구로 교체된다.
모바일 좁은 화면(`sm` 미만)에서는 페이지 제목과 겹쳐 보인다는 피드백으로 인사말
없이 닉네임만 노출한다.
