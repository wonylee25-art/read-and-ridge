- 미적용 시 위험: 로그인한 아무 사용자나 다른 사람의 책 기록을 볼 수 있음 (심각)
- **✅ 확인 완료 (2026-07-09)**: `books`, `hikes` 둘 다 RLS 켜져 있고 `auth.uid() = user_id` 정책 정상 적용됨

## 실행 가이드 (직접 따라하기) — 2026-07-09

### A. Supabase RLS 켜기 + 적용 확인

1. [supabase.com/dashboard](https://supabase.com/dashboard) → 프로젝트 선택 → 왼쪽 메뉴 **Table Editor** → `books` 테이블 클릭
2. 테이블 상단에 **RLS enabled** 토글이 꺼져 있으면 켜기 (초록색 = 켜짐)
3. 왼쪽 메뉴 **Authentication → Policies** (또는 **Database → Policies**)로 이동해 `books`, `hikes` 테이블에 정책이 있는지 확인
4. 정책이 없으면 **SQL Editor**에서 아래 SQL 실행 (`docs/database.md`에 있던 예시를 RLS 활성화까지 포함해 보완한 버전):

```sql
alter table books enable row level security;
alter table hikes enable row level security;

create policy "users can manage their own books"
  on books for all
  using (user_id = auth.uid())
  with check (user_id = auth.uid());

create policy "users can manage their own hikes"
  on hikes for all
  using (user_id = auth.uid())
  with check (user_id = auth.uid());
```

5. 확인 방법: Table Editor에서 다른 사용자로 로그인해 `books`를 조회했을 때 본인 것만 보이면 정상

### B. Google OAuth 설정

1. [Google Cloud Console](https://console.cloud.google.com/apis/credentials) → 해당 프로젝트 선택
2. **OAuth 동의 화면** 메뉴에서 게시 상태 확인 — "테스트中"이면 "프로덕션으로 게시" 또는 테스트 사용자 목록에 실제 계정 추가
3. **사용자 인증 정보 → OAuth 2.0 클라이언트 ID** 클릭 → **승인된 리디렉션 URI**에 배포 도메인 추가 (예: `https://<배포도메인>/auth/callback` 및 Supabase가 요구하는 콜백 URL)
4. Supabase 대시보드 → **Authentication → URL Configuration**에서 Site URL / Redirect URLs에도 배포 도메인 추가

### C. Vercel 환경변수 등록

1. [vercel.com](https://vercel.com) → 프로젝트 선택 → **Settings → Environment Variables**
2. 아래 값 등록 (Production 환경 체크):
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `NL_API_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY`
3. 값은 `.env.local` 파일 또는 Supabase 대시보드 → Project Settings → API에서 복사

### D. 로컬 빌드 확인

터미널에서 프로젝트 폴더로 이동 후:

```bash
npm run build
```

에러 없이 끝나면 배포 가능. (이번 세션 샌드박스는 네트워크 제한으로 끝까지 못 돌려봤음 — 로컬 확인 필수)

## 베타 기간에 확인할 것 — 2026-07-09

정식 오픈 전, 소수 테스터로 베타 운영하는 동안 아래 세 가지를 확인.

### 1. 기능이 실제로 도는지 (배포 도메인 기준)

구글 로그인 → 책 검색(ISBN 스캔 포함) → 등록 → 진행률 업데이트 → 완독 처리 → CSV 내보내기 → 회원 탈퇴까지 실제 배포된 URL에서 한 번씩 직접 눌러서 확인. 로컬에서 되는 것과 Vercel 배포본에서 되는 건 env 변수 누락, 리디렉션 URI 문제 등으로 다를 수 있음.

### 2. RLS가 실제로 작동하는지

테스터 2명 이상이 각자 로그인해서 서로 다른 사람 책 목록이 안 보이는지 직접 교차 확인. 정책 설정만 보고 넘어가지 말 것.

### 3. 터지는 지점 없는지

- Vercel 대시보드 → Functions/Logs에서 에러 발생 여부 (특히 NL API 호출, 회원탈퇴 흐름)
- 국립중앙도서관 API 호출량이 테스터들 검색으로 급증하는 건 아닌지
- 모바일에서 ISBN 스캔(카메라) 정상 동작하는지 — PC에서만 테스트했다면 특히 확인

테스터 피드백도 같이 받을 것. 위 세 가지 + 테스터 반응 확인되면 정식 오픈(Google OAuth 프로덕션 전환 포함) 진행.
