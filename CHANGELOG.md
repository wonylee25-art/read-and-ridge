# Changelog

산책또산책(Read & Ridge)의 버전별 변경 이력. 배포(push)할 때마다 여기에 기록하고,
`package.json`의 `version`과 `lib/version.ts`의 `LAST_UPDATED`도 같이 갱신할 것.

## 0.2.1 — 2026-07-12

- **KDC 산 색상 매핑**: 국립중앙도서관 서지정보(SEOJI) API 응답에서 `KDC`·`SUBJECT` 필드를
  추출해 책의 `kdc` 컬럼에 저장하도록 연결 (`/api/books/search`, `AddBookForm`, `addBook`).
  실측 확인 결과 `KDC` 필드는 CIP(사전 등록) 도서에서만 채워지고 대부분의 최신/자가출판
  도서는 비어 있어, `SUBJECT`(KDC 대분류 한 자리 숫자)를 1차 폴백으로 함께 사용.
  둘 다 없으면 기존처럼 `book.id` 해시 순환 배정으로 자연스럽게 대체됨(동작 변경 없음).
  "KDC가 산 색을 결정한다"는 핵심 컨셉의 남은 전제 조건이었던 부분(`docs/verification.md`
  ⚠️ 부분 6) 해소.

## 0.2.0 — 2026-07-11

- **책 추가 팝업**: 검색 결과 목록이 길어져도 팝업이 늘어나지 않고 목록 안에서만
  스크롤되도록 수정 (`max-h-72 overflow-y-auto`).
- **책 추가 팝업**: 저자가 여러 명이라 문자열이 아주 길어지는 경우, 자동 입력되는
  저자 값을 30자로 자르도록 수정 (`truncateAuthors`). 팝업 크기가 항상 고정되도록.
- **버전 표기**: 사이드바에 BETA 배지, 앱 버전(`vX.Y.Z`), 마지막 업데이트 날짜 표시 추가.
  버전 관리 체계 도입(`lib/version.ts`, 이 CHANGELOG).
