# Read & Ridge 인스타그램 게시물 9선 — 기능 소개형

기존 「빌드인퍼블릭 티저 9선」과는 다른 방향. 저 문서가 "만들고 있어요" 서사(narrative) 중심이었다면, 이 구조는 **문제 → 해결 → 기능 → 전환(CTA)** 순서로 앱을 직접적으로 설명하는 정보형 시리즈다. 카피 톤도 "저는... ㅋㅋ" 식 화자 서사 대신 헤드라인 + 짧은 설명 위주로 담백하게.

---

## 1. 문제 제기 — "이 책, 내가 이미 샀나?"

**목적**: 공감 훅. 앱 이름은 아직 밝히지 않는다.

**비주얼**: 책장 사진 또는 온라인 서점 장바구니 스크린샷 + 텍스트 오버레이

**헤드라인**: 이 책, 내가 이미 샀던가?

**캡션**
```
서점에서 책을 집었는데
어딘가 낯익다.

집에 있는 책인지 아닌지
검색해봐야 아는 그 순간,
다들 한 번쯤 겪어보셨을 거예요.

이 문제, 저희가 풀어보고 있습니다.
```

**해시태그**: #북스타그램 #독서기록 #중복구매 #만들고있어요

---

## 2. 해결 방식 소개 — Read & Ridge란

**목적**: 앱 이름과 핵심 컨셉 최초 공개

**비주얼**: 로고/타이틀 카드 + 월드맵 목업 한 컷

**헤드라인**: 소장 여부와 읽기 상태, 따로 관리합니다

**캡션**
```
Read & Ridge.
Read(독서)와 Ridge(산등성이)를 합쳤습니다.

이 책을 갖고 있는지, 그리고
어디까지 읽었는지 — 이 둘은 다른 질문이라
서로 다른 항목으로 따로 기록합니다.

읽은 만큼은 산의 높이로 쌓입니다.
```

**해시태그**: #Readandridge #독서기록앱 #픽셀아트 #앱소개

---

## 3. 기능 — 책 등록, 바코드 3초

**목적**: 진입 장벽 낮추기 (등록이 쉽다)

**비주얼**: 바코드 스캔 화면 녹화 or 손으로 폰 카메라 갖다 대는 클로즈업

**헤드라인**: 타이핑 없이, 3초

**캡션**
```
책 등록은 바코드 스캔 한 번이면 끝입니다.

제목, 저자, 쪽수까지
국립중앙도서관 데이터로 자동으로 채워져요.

직접 검색해서 등록하는 것도 물론 가능합니다.
```

**해시태그**: #ISBN #바코드스캔 #독서기록 #Readandridge

---

## 4. 기능 — 중복 구매 방지

**목적**: 1번 훅에 대한 명확한 답 제공

**비주얼**: 경고 팝업 UI 스크린샷 (화면 흔들림 모션 짧은 영상이면 더 좋음)

**헤드라인**: "또 산 책이 됩니다!"

**캡션**
```
이미 소장 중으로 등록한 책을
다시 등록하려 하면 바로 알려드립니다.

"또 산 책이 됩니다!"

1번 게시물에서 말씀드린 그 문제,
이렇게 막습니다.
```

**해시태그**: #중복구매방지 #책장정리 #독서기록앱

---

## 5. 기능 — 열람 상황 추적

**목적**: 소장 여부와 별개로 돌아가는 두 번째 축 설명

**비주얼**: 읽는 중 / 완독 / 잠시 멈춤 상태 3개 UI 나열 스크린샷

**헤드라인**: 빌린 책도, 안 산 책도 똑같이 기록됩니다

**캡션**
```
소장 여부와 상관없이
등록한 모든 책의 읽기 상태를 관리합니다.

읽는 중 / 완독 / 잠시 멈춤.
도서관 대출본도, 빌린 책도 예외 없이
같은 방식으로 산을 오릅니다.
```

**해시태그**: #독서기록 #독서상태 #Readandridge

## 6. 기능 — 월드맵, 책장이 산맥이 되는 화면

**목적**: 시각적 핵심 기능(가장 차별화되는 부분) 소개

**비주얼**: `worldmap-mockup-pastel.png` 전체 컷

**헤드라인**: 숫자 대신, 풍경으로

**캡션**
```
책의 쪽수는 산의 높이가 되고,
읽은 페이지만큼 캐릭터가 정상에 가까워집니다.

읽는 중인 책은 또렷한 전경 산으로,
아직 시작 안 한 책은 흐릿한 배경 설산으로 구분돼요.

내 독서 이력 전체가
하나의 산맥으로 쌓입니다.
```

**해시태그**: #픽셀아트 #독서기록앱 #WorldMap #Readandridge

## 7. 기능 — 산 색깔에 담긴 분류 체계

**목적**: 디테일 어필, 완성도 신뢰 구축

**비주얼**: 색이 다른 산들이 섞여 있는 컷 (문학=핑크, 과학=민트 등)

**헤드라인**: 산 색깔, 그냥 정해진 게 아닙니다

**캡션**
```
문학책은 라벤더, 과학책은 에메랄드.
책의 분류에 따라 산의 색이 자동으로 정해집니다.

분류명이나 숫자는 화면에 보이지 않아요.
색으로만 은은하게 드러납니다.

내 산맥의 색 분포만 봐도
어떤 책을 좋아하는지 알 수 있게 됩니다.
```

**해시태그**: #컬러시스템 #독서취향 #픽셀아트 #Readandridge

---

## 8. 기능 — 완등 세레모니와 정상석

**목적**: 감정적 페이오프, 가장 강력한 소재는 CTA 직전에 배치

**비주얼**: CLEAR! 애니메이션 영상 (있는 소재 중 최우선)

**헤드라인**: 다 읽으면, 정상석이 남습니다

**캡션**
```
완독하는 순간 CLEAR! 화면이 뜨고
캐릭터가 정상에서 양팔을 번쩍 듭니다.

그 자리엔 정상석이 솟아오르고
책 제목과 완독일이 새겨집니다.

완독한 책들만 모아보는
정상석 갤러리도 마이페이지에 따로 있어요.
```

**해시태그**: #완독 #독서인증 #8비트 #Readandridge #완등기록

## 9. CTA — 베타 테스터 모집

**목적**: 지금까지 소개한 기능을 요약하고 행동 유도

**비주얼**: 1~8번 스크린샷/영상 모아 만든 하이라이트 릴스

**헤드라인**: 먼저 써보실 분을 찾습니다

**캡션**
```
지금까지 소개해드린 기능들,
정식 오픈 전 소규모 베타로 먼저 열어보려 합니다.

- 화면은 아직 다듬어지는 중이고
- 오류 제보에 답변이 조금 늦을 수도 있어요

내 책장이 산맥이 되는 걸
가장 먼저 보고 싶으신 분,
댓글이나 DM으로 알려주세요 🏔️
```

**해시태그**: #베타테스터모집 #독서기록앱 #Readandridge #북스타그램

---

## 참고 — 이 구조의 특징

- **AIDA 흐름**: 1(문제) → 2(해결/컨셉) → 3~8(기능 6개) → 9(전환)
- 기존 티저 문서와의 차이: 화자의 개인 서사("저는 산은 안 오르는데 ㅋㅋ") 대신, 헤드라인 + 담백한 설명형 캡션으로 통일
- 3~8번은 순서를 바꿔도 무방한 독립 기능 모듈이라, 촬영/개발 진행 상황에 맞춰 유동적으로 배치 가능
- 8번(완등 세레모니)은 가장 임팩트 있는 소재이므로 CTA 직전 자리를 유지하는 걸 권장
